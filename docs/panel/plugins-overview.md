# Plugins overview

This page explains plugin management in one place, then links to each plugin guide.

## Plugin lifecycle

Typical plugin workflow in the Panel:

1. Open your Space.
2. Review available plugins.
3. Enable and configure the plugin.
4. Verify plugin behavior in content workflows.
5. Update or disable when no longer needed.

## Good management practices

- Enable plugins per use case, not all at once.
- Test plugin behavior in a staging Space before production use.
- Document plugin owner and configuration in your team.
- Review plugin list regularly and disable unused plugins.

## Plugin categories in docs

- Content workflow plugins (for example Slug, Live Preview, Content Preview)
- Integrations (for example Netlify, Gatsby Cloud, Cloudflare Stream)
- Editorial helpers (for example Color Picker, Kanban Board)
